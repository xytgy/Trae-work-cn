# 像素地牢：分支路线 Roguelite 优化设计

**日期：** 2026-07-02
**状态：** 设计草案

---

## 一、目标

解决游戏前期"无聊"的问题：
- 战斗房比例偏低（40%），非战斗房无敌人，导致节奏空洞
- 敌人种类解锁太慢，前期组合单一
- 玩家对路线无控制，缺少 roguelike 的决策感

---

## 二、房间结构

### 2.1 固定序列

单元测试 1（起始战斗）→ 战斗房 2 → **分支选择** → 特殊房 → 战斗房 3 → **精英房** → **Boss房**

共 **6 间房**（当前 4 间），战斗房占比从 40% 提升到约 50%。

### 2.2 分支选择（三选一）

第 2 间战斗房清空后，触发 `GAME_STATE.ROUTE_SELECT`，玩家从以下 3 个选项中选 1 个：

| 选项 | 类型 | 描述 |
|------|------|------|
| 🏆 精英房 | ELITE | 精英怪 + 小兵，高奖励（金宝箱） |
| 🛒 商店房 | SHOP | 使用金币购买道具（无战斗） |
| 💤 休息房 | REST | 回血喷泉，免费恢复 2 点生命（无战斗） |

- 每局每种特殊房 **只出现一次**（不会重复抽到 2 间商店）
- 选项 UI 用 Canvas 绘制，鼠标点击或数字键 1/2/3 选择
- 选择时游戏暂停，选完后 `initRoom(selectedType)` 创建对应房间

### 2.3 各房间类型概览

| 房间 | 战斗? | 描述 |
|------|-------|------|
| 起始战斗房 | ✅ 4 敌 | 史莱姆 + 蝙蝠 |
| 战斗房 2 | ✅ 4-5 敌 | 史莱姆 + 蝙蝠 + 骷髅 |
| 战斗房 3 | ✅ 5-6 敌 | 全员混编 |
| 精英房 | ✅ 精英 + 3 小兵 | 强化精英 + 金宝箱奖励 |
| 商店房 | ❌ | 商人购物 |
| 休息房 | ❌ | 回血喷泉 |
| Boss 房 | ✅ Boss | 最终战 |

## 三、敌人丰富度

### 3.1 敌人类型解锁提前

| 房间 | 当前 | 改后 |
|------|------|------|
| 房间 1 | slime, skeleton | slime, skeleton, **bat** |
| 房间 2 | slime, bat, skeleton | slime, bat, skeleton, **ghost, archer** |
| 房间 3 | slime, bat, skeleton, archer | 全员可用 |
| 房间 4+ | 全员 + mage, bomber | 全员 |

### 3.2 非战斗房守卫

为宝箱房和商店房分别引入守卫逻辑（在非Boss的特殊房间中添加 1-2 个守护敌人）。清理守卫后才能打开宝箱/进入商店。

| 房间类型 | 守卫数量 |
|----------|---------|
| 宝箱房 | 1-2 个（随房间递增） |
| 商店房 | 1 个 |
| 陷阱房 | 1 个 |
| 休息房 | 0 个 |

### 3.3 敌人数量提升

```
当前公式：baseCount = 3 + floor(roomIndex * 0.5)
新公式：  baseCount = 4 + floor(roomIndex * 0.8)

实际表现：
  房间 1: 4 只（当前 3）
  房间 2: 4 只（当前 3）
  房间 3: 5 只（当前 4）
  房间 4: 6 只（当前 4）
  精英房: 精英 + 3 小兵
```

## 五、不做的范围

- ❌ 不改为完整地图（如 Slay the Spire 式），保持轻量级三选一
- ❌ 不改动 Boss 房设计
- ❌ 不新增敌人类型（只改变解锁顺序和生成数量）
- ❌ 不做武器升级/遗物合成等系统（已超出本次范围）
