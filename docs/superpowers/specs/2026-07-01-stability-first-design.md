# 像素地牢：枪战冒险 — 先稳后优设计规格

## 概述

**目标**: 将现有游戏从"部分可用的 Demo"提升到"全流程稳定的可发布产品"

**核心原则**:
- 稳定性优先：先确保全流程无崩溃，再添加新功能
- 渐进式集成：模块化框架逐步替换，每次替换后测试
- 质量保证：每个修复和优化都有测试验证

## 第0阶段：全流程稳定性修复

### 目标
确保游戏从主菜单→角色选择→难度选择→战斗→Boss战→死亡/通关→重新开始的全流程无任何崩溃。

### 风险点清单与修复方案

#### 风险点1：Boss战崩溃（最高优先级）

**风险描述**: Boss三阶段攻击（激光、冲锋、回血）、阶段转换、召唤小怪等路径未充分测试，可能存在引用错误、参数传递错误、状态机错误。

**修复方案**:

1. **Phase 1 - 散弹攻击**:
   - 检查 `spreadshotAttack` 方法中 `gameLogic` 参数是否正确传递
   - 验证子弹创建时 `PARTICLES.TYPES` 配置是否完整
   - 添加空值检查：`gameLogic && gameLogic.bullets && ...`

2. **Phase 2 - 狂暴状态**:
   - 检查冲锋逻辑中 `player` 引用是否在攻击期间保持有效
   - 验证召唤小怪时 `setTimeout` 的 `cancelToken` 机制是否有效
   - 添加 `isCharging`、`chargeWarning` 状态的边界检查

3. **Phase 3 - 激光攻击**:
   - 检查激光渲染时 `ctx.createRadialGradient` 参数是否合法
   - 验证激光计时器逻辑是否正确（预警→发射→冷却）
   - 添加回血蓄力期间的伤害检测和中断机制

4. **阶段转换**:
   - 确保 `isPhaseTransitioning` 标志正确设置和清除
   - 验证阶段转换期间 Boss 是否无敌
   - 添加阶段转换特效的资源清理

**预期改动文件**:
- `js/boss.js` — 修复三阶段逻辑、添加空值检查、状态机优化
- `js/game.js` — 确保 Boss 更新时传递正确参数

**测试用例**:
- TC-Boss-001: 普通模式下通关 Boss（不使用任何技能）
- TC-Boss-002: 使用冲刺技能穿过 Boss 激光
- TC-Boss-003: 在 Boss 回血蓄力期间打断
- TC-Boss-004: 连续两次挑战 Boss（第一次失败，第二次成功）

#### 风险点2：技能系统崩溃

**风险描述**: skills.js 包含27个技能类，1170行代码，几乎未测试过。可能存在：
- 引用未定义的全局变量（`particleSystem`、`inputManager`）
- 访问 `this.owner` 的属性时未做空值检查
- 技能执行时参数传递错误

**修复方案**:

1. **全局变量空值保护**:
   - 在所有技能的 `execute()` 和 `update()` 方法开头添加 guard clauses
   - 示例：`if (!particleSystem || !this.owner) return;`

2. **技能状态管理**:
   - 确保每个技能的状态标志（如 `isDashing`、`isActive`）正确初始化和重置
   - 添加技能释放前的条件检查（冷却时间、怒气值等）

3. **粒子系统调用**:
   - 检查所有 `particleSystem.createParticle()` 调用的参数数量和类型
   - 验证 `PARTICLES.TYPES` 和 `PARTICLES.KIND` 配置是否完整

4. **输入系统调用**:
   - 检查 `inputManager.getMouseWorldPosition()` 返回值是否有效
   - 添加鼠标位置为空时的降级处理

**预期改动文件**:
- `js/skills.js` — 添加空值检查、状态保护、参数验证
- `js/skill.js` — 优化基础技能类的错误处理

**测试用例**:
- TC-Skill-001: 每个主动技能至少释放一次，无崩溃
- TC-Skill-002: 被动技能在战斗中正确触发
- TC-Skill-003: 技能冷却期间重复点击无异常
- TC-Skill-004: 连续释放多个不同技能组合

### 第0阶段通用修复策略

#### 1. 全局错误捕获

在 `main.js` 的游戏循环中添加全局错误捕获：

```javascript
gameLoop(currentTime) {
    if (!this.running) return;
    
    try {
        // 游戏更新逻辑
        if (gameState.isPlaying()) {
            this.gameLogic.update(clampedDelta);
            // ...
        }
    } catch (error) {
        console.error('[GAME LOOP ERROR]', error);
        // 记录错误但不中断游戏循环
        this.handleGameError(error);
    }
    
    try {
        // 游戏渲染逻辑
        this.gameLogic.render();
        uiManager.update(clampedDelta);
        uiManager.render();
    } catch (error) {
        console.error('[RENDER ERROR]', error);
    }
    
    this.animationFrameId = requestAnimationFrame(this.gameLoop.bind(this));
}
```

#### 2. 空值检查规范

所有函数内部引用的变量必须进行空值检查：

```javascript
// 错误方式
function doSomething() {
    gameLogic.player.x += 10;
}

// 正确方式
function doSomething() {
    if (!gameLogic || !gameLogic.player) return;
    gameLogic.player.x += 10;
}
```

#### 3. 参数类型检查

所有从配置/JSON读取的值在参与数学运算前确保是 number 类型：

```javascript
const damage = parseFloat(config.damage) || 1;
const speed = parseInt(config.speed) || 1;
```

#### 4. Canvas 方法参数验证

所有 `ctx.*` 调用的参数必须是合法数字：

```javascript
if (typeof x === 'number' && typeof y === 'number') {
    ctx.drawImage(image, x, y);
}
```

## 第1阶段：分支路线方案落地

### 目标
完善精英/商店/休息房间的切换功能，确保路线选择系统稳定可靠。

**需求**: 战斗房间清除后，显示路线选择界面（精英/商店/休息），玩家选择后进入对应房间。

**实现要点**:
- 确保路线选择界面在正确时机触发（所有敌人清除后）
- 验证路线选择后房间类型正确设置
- 添加路线选择的键盘快捷键支持（1=精英, 2=商店, 3=休息）

**测试用例**:
- TC-Route-001: 战斗房间清除后显示路线选择
- TC-Route-002: 选择精英路线进入精英房间
- TC-Route-003: 选择商店路线进入商店并购买物品
- TC-Route-004: 选择休息路线恢复生命值

#### 2. 精英房间

**需求**: 精英房间包含强化敌人，击败后获得更好的掉落。

**实现要点**:
- 确保精英敌人的属性正确计算（基于难度和房间进度）
- 验证精英敌人掉落率和掉落物品类型
- 添加精英房间的视觉标识（房间边框颜色、敌人外观）

**测试用例**:
- TC-Elite-001: 精英房间敌人数量和属性正确
- TC-Elite-002: 击败精英敌人后获得稀有掉落
- TC-Elite-003: 精英房间失败后返回主菜单

#### 3. 商店房间

**需求**: 商店房间包含可购买的物品（武器、药水、遗物）。

**实现要点**:
- 确保商店物品生成逻辑正确（基于当前进度）
- 验证购买流程（金币扣除、物品获取）
- 添加商店界面的交互反馈（悬停效果、购买动画）

**测试用例**:

- TC-Shop-002: 购买物品后金币正确扣除
- TC-Shop-003: 金币不足时无法购买，有提示

#### 4. 休息房间

**需求**: 休息房间包含回血喷泉，玩家接触后恢复生命值。

**实现要点**:
- 确保回血喷泉的碰撞检测正确
- 验证回血效果（单次恢复量、恢复上限）
- 添加回血时的视觉和音效反馈

**测试用例**:
- TC-Rest-001: 接触回血喷泉恢复生命值
