**日期：** 2026-07-02
**状态：** 设计草案

## 一、整体关卡层级

### 1.1 三级结构

大关 → 小关 → 房间

- **共 4 个大关**，按顺序推进
- **每个大关包含 5 个小关**，标记为 1-1 ~ 1-5、2-1 ~ 2-5 等
- **前 4 个小关**（1-1 ~ 1-4）终点为传送门房间
- **第 5 小关**（1-5）终点为 Boss 房间
- 每清完一小关，通过传送门进入全新生成的 5×5 地图

### 1.2 大关主题

每个大关有一套独立的主题配置（场景颜色、敌人池、Boss、难度倍率），每局随机抽取：

| 大关 | 可选主题 | 难度倍率 |
|------|---------|---------|
| 第 1 大关 | 森林 / 冰原 / 遗迹 | 1.0 |
| 第 2 大关 | 城堡 / 墓穴 / 沼泽 | 1.3 |
| 第 3 大关 | 火山 / 深渊 / 神庙 | 1.6 |
| 第 4 大关 | 虚空 / 王座 / 终末 | 2.0 |

### 1.3 状态继承

- 玩家的**血量、金币、武器、技能、遗物**等状态在各大关之间继承
- **只有死亡 / 重开**才会清零
- 每小关不重置玩家状态

## 二、单小关地图布局

### 2.1 5×5 网格

- 所有房间分布在 **5×5 = 25 个网格位置**中
- 房间可向 **上下左右四个方向**相邻连接
- 禁止纯横向一字排列

### 2.2 主路径（固定长度）

**1~4 小关（非 Boss 关）：**

```
初始房间 → 基础小怪房(1) → 基础小怪房(2) → 传送门房间
主路径长度：4 个房间（含初始和终点）
```

**第 5 小关（Boss 关）：**

```
初始房间 → 基础小怪房(1) → 基础小怪房(2) → 基础小怪房(3) → Boss 房间
主路径长度：5 个房间（含初始和终点）
```

### 2.3 分支房间

- 每个**基础小怪房**可随机向侧面（上/下/左/右）连接 **0~1 个分支房间**
- 分支房间类型：宝箱房、特殊事件房、额外小怪房
- 分支全部为**可选探索**，不影响主线推进
- **初始房间、传送门房间、Boss 房间**仅连接 1 个主路径房间，不生成分支

### 2.4 生成流程

```
1. 初始化 5×5 空网格
2. 放置初始房间在网格中心附近 (2,2) 或 (2,3)
3. 随机方向延伸主路径（固定长度，保证连通）
4. 在主路径的基础小怪房侧面随机生成分支
5. 为分支房间分配随机类型
6. 校验全图连通性，若有孤立房间则重新生成
7. 标记每个房间在网格中的坐标 (gridX, gridY) 和世界坐标
```

## 三、房间类型

| 类型 | 出现位置 | 战斗? | 行为 |
|------|---------|------|------|
| 初始房间 | 主路径起点 | ❌ | 玩家出生点，无敌人。入口关闭（起点），出口通往主路径第 1 个小怪房 |
| 基础小怪房 | 主路径中间 | ✅ | 踏入后关门锁死，清完所有敌人后门自动开启。掉落基础金币 |
| 额外小怪房 | 分支 | ✅ | 敌人数量+1~2，强度略高，奖励更丰厚 |
| 宝箱房 | 分支 | ❌ | 无战斗，中央放宝箱，产出武器/药水/金币 |
| 特殊事件房 | 分支 | ❌ | 小地图显示黄色「!」标记，随机触发（雕像附魔/工匠升级/许愿池/商人） |
| 传送门房 | 1~4 小关终点 | ❌ | 无敌人，中央放置传送门，触碰后进入下一小关 |
| Boss 房 | 第 5 小关终点 | ✅ Boss | 踏入后关门，击败 Boss 后出现传送门 + 金色宝箱 |

## 四、房间连接与门禁

### 4.1 相邻规则

- 相邻房间**共用一面墙体**
- 墙体中央设置**门**，无独立走廊/过渡通道
- 所有房间尺寸统一，门的位置在两房间相邻墙体的中点

### 4.2 门禁逻辑

| 房间类型 | 入口行为 | 出口行为 |
|---------|---------|---------|
| 初始房 | 默认关闭（起点） | 默认开启 |
| 基础小怪房 | 踏入后关闭锁死 | 清空敌人后开启 + 解锁通往后续房间的门 |
| 额外小怪房 | 踏入后关闭锁死 | 清空敌人后开启 |
| 宝箱房 | 默认开启 | 默认开启 |
| 特殊房 | 默认开启 | 默认开启 |
| 传送门房 | 默认开启 | — |
| Boss 房 | 踏入后关闭锁死 | Boss 死亡后开启 |

### 4.3 相机与渲染

- **相机固定**：进入房间后，相机立即锁定到房间中心（硬切换，无平滑过渡）
- **单屏覆盖**：每个房间尺寸固定，单屏完整展示，不需要卷轴滚动
- **迷雾机制**：
  - 离开初始房后小地图可见
  - 仅**已进入的房间**和**已连通的已清怪房间**完整显示
  - 未探索区域保持隐藏

## 五、房间尺寸

### 5.1 三种规格

参照元气骑士，房间可有不同尺寸但都在单屏内：

| 规格 | 尺寸（像素） | 玩家视野 | 适配场景 |
|------|------------|---------|---------|
| 小型 | 640×480 | 完整覆盖 | 初始房、宝箱房、传送门房 |
| 中型 | 800×600 | 完整覆盖 | 基础小怪房、特殊房 |
| 大型 | 960×720 | 完整覆盖 | Boss 房、精英房 |

所有尺寸**高度方向居中于画布，宽度方向居中或左对齐**，相机始终对齐房间中心。

### 5.2 房间编号与门编号

每个房间有 4 个门位置（上/下/左/右），用 bitmask 表示：
```
DOOR_TOP    = 1 << 0  (1)
DOOR_BOTTOM = 1 << 1  (2)
DOOR_LEFT   = 1 << 2  (4)
DOOR_RIGHT  = 1 << 3  (8)
```

例如：一扇上门 + 一扇右门 = `DOOR_TOP | DOOR_RIGHT = 9`

        this.gridX = gridX;       // 网格 X (0-4)
