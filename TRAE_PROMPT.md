# 像素地牢：枪战冒险 — 项目修复与优化提示

## 项目简介

一个 HTML5 Canvas 原生 JavaScript 像素风地牢射击游戏。玩家在随机生成的地牢中闯关，击败 Boss 通关。

**运行方式：** 直接在浏览器打开 `index.html`

## 核心目标（按优先级排序）

### 一、稳定性（不能让游戏崩溃/卡死/异常退出）

这是最高优先级。必须确保：

1. **全流程测试**：从主菜单 → 角色选择 → 难度选择 → 游戏进行 → 进入不同房间 → 战斗 → 商店/宝箱交互 → Boss战 → 死亡/通关 → 重新开始，全流程不出现任何异常
2. **变量引用检查**：每个函数内部引用的变量必须已声明。特别检查：
   - `PARTICLES.TYPES.*` 配置是否完整
   - `config` 变量在每个粒子效果函数中是否有声明
   - `UI_COLORS.*`、`BUILD_TYPE` 等常量是否定义
3. **参数传递检查**：所有函数调用传参顺序是否与函数签名匹配
4. **Canvas 方法检查**：所有 `ctx.*` 调用（如 `createRadialGradient`、`drawImage`）的传入参数必须是合法数字
5. **字符串转数字**：所有从配置/JSON读取的值在参与数学运算前确保是 number 类型
6. **对象空值检查**：访问嵌套属性前做 guard（如 `gameLogic?.particles?.push`）
7. **游戏循环异常捕获**：在 gameLoop 的主更新/渲染部分加 try-catch，防止单一异常终止整个循环
8. **性能兜底**：粒子数量超过上限时自动裁剪、子弹数量超过上限时回收

### 二、代码优化

1. **双系统合并**：`audioManager`（audio.js）和 `SoundManager`（soundManager.js）都做音频，建议合并为一个。`soundManager` 功能更全（有 20+ 种音效），保留它，弃用 `audioManager`
2. **废弃代码删除**：
   - `index.html` 中移除 `modules/` 下所有 8 个文件的 script 引用（它们从未被实例化）
   - 移除 `config/` 目录的 JSON 文件引用（从未被加载）
   - 移除 `weapon.js`、`weaponPistol.js`、`weaponLightning.js` 等废弃武器类的 script 引用（子弹创建在 game.js 中硬编码）
3. **大文件拆分**：
   - `game.js`（4181行）→ 至少拆出碰撞检测、粒子效果生成、房间初始化
   - `ui.js`（3081行）→ 按界面拆分（菜单UI、HUD、设置、背包）
4. **去除全局变量污染**：所有类实例（game、gameState、renderer、uiManager 等）应该用 const/let 声明在文件作用域，不要泄漏到 window
5. **HTML script 标签整理**：清理加载顺序注释，移除未使用文件的引用

### 三、精致程度（Polish）

1. **视觉反馈增强**：
   - 击中敌人时，伤害数字浮动（已实现但颜色/动画可以更好）
   - 击杀敌人时屏幕微震（已实现但强度太小）
   - 拾取武器时的"时间停止"特效（已有代码但可能未触发）
   - 拾取金币时飘字 "+5G"
2. **UI 一致性**：
   - 所有界面字体统一（现在部分用 Arial，部分用 Courier New，部分用 Microsoft YaHei）
   - 按钮悬停效果统一（有些有缩放手感，有些没有）
3. **粒子效果**：
   - 环境灰尘粒子太少，房间太安静
   - 火焰喷射器的火焰粒子可以更密集
   - Boss 阶段转换时缺少特效
4. **动画平滑度**：
   - 传送门打开时有倒计时，但没有动画过渡
   - 房间切换时直接跳转，缺少淡入淡出

### 四、用户体验优化

1. **操作手感**：
   - 当前加速度 1.5 / 最大速度 4.8，可以再微调到 1.8 / 5.2 让手感更跟手
   - 受伤后无敌帧闪烁太急促，可以调慢闪烁间隔
2. **引导与提示**：
   - 新手第一间房在地面画移动提示箭头
   - 武器切换在屏幕中央显示大图标提示（"切换到：🔥火焰喷射器"）
   - 弹药耗尽时自动切回手枪加文字提示
3. **信息可见性**：
   - 怒气条的数值太小，改为百分比+数字显示
   - Boss 血条显示阶段名称（如 "阶段 2 - 狂暴"）
   - 当前武器弹药在玩家头顶显示小弹药图标
4. **容错处理**：
   - localStorage 读写失败时不影响游戏运行
   - AudioContext 创建失败时静默降级，不阻塞初始化
   - 游戏在后台标签页时自动暂停
5. **可访问性**：
   - 支持游戏手柄（已有代码框架但未完全接入）
   - 设置界面增加"重置为默认"按钮

## 工作流程

### 阶段 1：设计文档（必须先做）
阅读整个项目的所有核心文件（尤其是 constants.js、game.js、main.js、state.js、input.js、ui.js、renderer.js），写出详细的设计文档，包括：
- 当前架构分析
- 稳定性风险点清单
- 每个优化项的改动方案
- 不做的内容清单

### 阶段 2：稳定性修复
按阶段 1 的风险点清单逐个修复，每个修复后做一次浏览器的全流程测试（菜单→战斗→死亡/通关→重新开始）

### 阶段 3：代码优化
合并音频系统、删除死代码引用、拆分大文件

### 阶段 4：精致度 + UX
视觉增强、UI 一致性、操作引导优化

### 阶段 5：完整测试
至少完整通关 3 次，每次使用不同角色和不同武器组合

## 约束条件

- 所有代码使用原生 JavaScript（ES6+），无框架依赖
- 画布使用 Canvas 2D，不引入 WebGL
- 不添加外部 npm 包或 CDN 资源
- 所有音效使用 Web Audio API 合成，不需要音频文件
- 保持像素风格美术方向
