# 走廊位置修正计划

## 问题分析

走廊被渲染在房间内部而不是房间外部，根本原因是房间背景的门切口太大。

**错误代码位置**: `room.js` 第 641-656 行

```javascript
const corridorLength = this.width / 2;  // 错误：走廊长度 = 房间宽度的一半

if (roomNode.hasDoor(DOOR.RIGHT)) {
    ctx.fillRect(this.width - corridorLength, centerY - halfDoorWidth, corridorLength, doorWidth);
    // 这会从房间边缘向内切一半的房间宽度，形成巨大的透明洞
}
```

**问题效果**:
- 门切口从房间边缘延伸到房间中心（400像素）
- 走廊虽然正确地位于房间边界外，但看起来像是在房间内部
- 因为房间背景有一个巨大的透明区域，走廊透过这个区域显示出来

## 解决方案

### 方案一：缩小门切口（推荐）
将门切口改为仅在房间边缘切出一个小门，走廊保持在房间外部。

**修改内容**:
1. 将 `corridorLength` 改为 `LEVELS.WALL_THICKNESS`（墙壁厚度）
2. 确保门切口只移除墙壁部分，不影响房间内部

### 方案二：完全移除门切口
让走廊覆盖在房间墙壁之上，不修改房间背景。

**修改内容**:
1. 移除 `destination-out` 切门逻辑
2. 走廊渲染在房间之后，覆盖墙壁

- **低风险**: 修改门切口长度只会影响视觉效果，不会改变游戏逻辑
- **兼容性**: 此修改与现有代码兼容，不需要修改其他文件

4. 战斗模式下玩家被限制在房间内
