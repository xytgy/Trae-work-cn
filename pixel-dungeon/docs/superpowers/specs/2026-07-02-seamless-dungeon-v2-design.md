# 无缝地牢系统 V2 - 产品需求文档

## Overview
- **Summary**: 实现真正的无缝地牢地图系统，玩家可以在房间和走廊之间自由移动，战斗时相机锁定在房间范围内。
- **Purpose**: 打造真实的地牢探索体验，玩家可以自由漫步，不再有瞬移感。
- **Target Users**: Pixel Dungeon: Gunfight Adventure 游戏玩家

## Goals
- 实现真正的无缝世界，所有房间和走廊在连续坐标中
- 房间之间有物理走廊，玩家可以步行通过
- 战斗模式相机锁定房间，探索模式相机跟随玩家
- 代码结构清晰，易于维护和扩展

## Non-Goals (Out of Scope)
- 走廊敌人（留待后续迭代）
- 复杂地形生成
- 多层地牢

## Architecture

### 核心架构
```
World (世界)
├── DungeonLevel (地牢层级)
│   ├── RoomNode[] (房间节点 - 坐标、连接关系)
│   └── Corridor[] (走廊 - 连接房间)
├── GameObjects (游戏对象)
│   ├── Player (玩家 - 世界坐标)
│   ├── Enemy[] (敌人 - 世界坐标)
│   ├── Bullet[] (子弹 - 世界坐标)
│   └── Particle[] (粒子 - 世界坐标)
└── Camera (相机)
    ├── Explore Mode (探索模式 - 跟随玩家)
    └── Battle Mode (战斗模式 - 锁定房间)
```

### 坐标系统
- **世界坐标原点**: (0, 0) 在左上角
- **房间尺寸**: ROOM_WIDTH x ROOM_HEIGHT
- **走廊宽度**: DOOR_SIZE
- **走廊长度**: ROOM_WIDTH / 2 (水平) 或 ROOM_HEIGHT / 2 (垂直)

## Functional Requirements

### FR-1: 世界坐标系统
- 所有游戏对象使用世界坐标
- 房间按网格排列：(gridX * ROOM_WIDTH, gridY * ROOM_HEIGHT)
- 走廊连接相邻房间

### FR-2: 走廊系统
- 相邻房间之间自动生成走廊
- 走廊有墙壁（顶部/底部或左侧/右侧）
- 走廊有地砖渲染
- 走廊墙壁有碰撞检测

### FR-3: 相机系统
- **探索模式**: 相机跟随玩家，可以看到走廊和相邻房间
- **战斗模式**: 相机锁定在当前房间范围内
- 自动检测切换：有敌人进入战斗模式，无敌人进入探索模式

### FR-4: 玩家移动
- 玩家可以自由移动通过走廊
- 走廊墙壁阻止玩家穿过
- 房间边界在战斗模式下限制玩家

### FR-5: 房间切换
- 玩家进入新房间时自动加载房间内容
- 已访问房间状态保留（传送门等）
- 房间切换平滑，无瞬移感

### FR-6: 战斗触发
- 玩家进入有敌人的房间时切换战斗模式
- 战斗模式下相机锁定，玩家被限制在房间内

### FR-7: 传送门系统
- Boss房间或传送门房间清除后生成传送门
- 传送门有动画效果和碰撞检测
- 进入传送门切换到下一关

## Implementation Plan

### Phase 1: 世界坐标系统
1. 修改 RoomNode 添加世界坐标属性
2. 创建 Corridor 类
3. 修改 DungeonLevel 生成走廊

### Phase 2: 相机系统
1. 修改 Camera 类添加双模式
2. 添加相机锁定边界逻辑

### Phase 3: 房间渲染
1. 修改 Room 类使用世界坐标渲染
2. 渲染所有已访问房间
3. 渲染走廊

### Phase 4: 玩家移动和碰撞
1. 修改玩家位置使用世界坐标
2. 添加走廊墙壁碰撞检测
3. 修改边界限制逻辑

### Phase 5: 战斗和传送门
1. 修改战斗触发逻辑
2. 修改传送门生成和碰撞检测
3. 修改关卡切换逻辑

## Success Criteria
- 玩家可以自由通过走廊走到相邻房间
- 战斗模式下玩家被限制在房间内
- 传送门正常工作，进入后切换关卡
- 游戏运行流畅，无明显卡顿
- 代码结构清晰，易于维护
